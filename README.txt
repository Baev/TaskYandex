Мое решение задания №2.
Задача - написать инструкцию для тестировщика страниц.

Используемые технологии:
1) WebDriver анализа содержимого страницы
2) Система сборки Maven
3) Система контроля версий Git

Описание решения
На самом деле поставленная задача не так проста. 
Какие ошибки могут быть:
	- Ошибки верстки. 
		Незакрытый тег и из-за этого непредсказуемый результат.
		Или того хуже, ошибка в написании какого-нибудь атрибута тега. Что с этим делать - непонятно.
		Будем считать, что таких ошибок нету. Наверняка, есть написанная программа проверки корректности верстки.
		Можно ее прикрутить. 
		
	- Битые ссылки.
		Ищем видимые ссылки, созданные с помощью тега a.
		Про псевдоссылки забываем. Непонятно, что с ними делать.
		Выпадающие при наведении мыши меню - стараемся обработать. 
		Меню, появляющиеся при клике - нет. Там сплошной javscript.
		
	- Ошибки форм. 
		Будем работать с тегами input, button, textarea, находящимися внутри тега form. Без тега form - это
		ошибки верстки. Обработать несложно, решил не загромождать решение.
		Типы, добавленные для тега input в html5 обрабатывать не будем.
		Также я не научился искать текст кнопки найти на yandex.ru и названия radio элементов. 
		Текст, вводимый в текстовые поля решил не генерировать - загромождает результат + не несет 
		никакой смысловой нагрузки.
		Для тестирования использую решение задачи прошлой. Состовляю массив Ai, генерирую тесты, результат перевожу
		в инструкцию для тестировщика. 
		
	- Отсутствующие картинки. 
		Непонятно, как это определять. Я так и не научился. Покликаю по картинкам, на которых есть ссылки.
		Опять же, не по всем. Еще есть псевдокартинки, которые не научился находить. 
		
	- Ошибки содержания.
		Тут уж без человека не обойтись. Наверное.
